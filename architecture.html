🤖 GOOGLE AI — Gemini APIs
🗣
Gemini Live API
gemini-3.1-flash-live-preview
Bidirectional audio stream. Handles STT → LLM → TTS in a single pipeline. Provides inputAudioTranscription + outputAudioTranscription. Supports function calling mid-conversation.
24 kHz PCM Real-time 8 Voices
📦
Gemini Files API
genai.files.upload / list / delete
Temporary cloud storage for documents (48-hr lifetime). Supports PDF and text/plain. Each file tagged with sessionId::filename in displayName for per-session filtering.
48-hr TTL PDF + TXT
🧠
Gemini Flash (RAG)
gemini-2.0-flash · generateContent
Used for document search. Receives the full document as fileData and extracts verbatim relevant passages for the user's query. Runs in parallel across all uploaded files.
Document QA 1M ctx window

🌍 EXTERNAL DATA SOURCES
📂
PDF Upload

Direct file upload from the user's device. Max 100 MB. Processed immediately and stored in Gemini Files API.
🔗
Web URL Import
node fetch + cheerio
Fetches any public web page. Removes nav/footer/scripts. Extracts readable text from article/main/body. Stores as text/plain (up to 500 KB).
Google Drive
googleapis · Application Default Credentials
User shares a file with the Cloud Run service account. Backend downloads via Drive API using ADC. File ID extracted from shareable link URL.
🪣
Google Cloud Storage
@google-cloud/storage · ADC
User provides gs://bucket/path. Cloud Run service account reads the object using ADC. Downloaded and uploaded to Gemini Files API.
Key Data Flows
🗣 Voice Conversation
1
User clicks Start session — mic auto-activates, AudioContext created during gesture
2
Browser captures mic at 16 kHz PCM, sends binary WebSocket frames to Cloud Run
3
Node.js proxies audio → Gemini Live via sendRealtimeInput()
4
Gemini streams back audio + transcript chunks. Server relays as JSON messages
5
Browser plays 24 kHz PCM via AudioContext with 50 ms lookahead buffer
🔍 Document RAG Search
1
User asks a question about their documents (voice or text)
2
Gemini Live decides to call search_documents function with a query
3
Node.js fetches all active files for this session from Gemini Files API
4
Fans out parallel generateContent calls — one per file with fileData
5
Relevant passages returned to Gemini Live as tool response → spoken to user
📄 Document Import
1
User drops PDF / pastes URL / Drive link / GCS path
2
For URL: fetch → cheerio HTML parse → extract text; For Drive/GCS: download via ADC
3
Buffer uploaded to Gemini Files API with sessionId::filename displayName
4
File transitions to ACTIVE state (usually instant). Returns fileUri for RAG
5
File listed in sidebar. Automatically expires after 48 hours
🕓 Session History
1
Every message is stored in React state as TranscriptEntry[]
2
On disconnect, transcript auto-saved to localStorage with generated title
3
History tab lists up to 50 sessions sorted newest-first
4
Click any session to restore read-only transcript view in chat area

Deployment
🌐 Live URL
https://talk-to-doc-api-1085727639300
.us-central1.run.app
Single Cloud Run service serving both API and SPA from one container. Zero cold-start overhead on active sessions.
🏗 Build Pipeline
Cloud Build → Artifact Registry → Cloud Run
Docker multi-stage: builds React SPA + TypeScript API, copies web/dist into the runner image for Express static serving.
🔑 Service Account
Cloud Run SA has Drive API + GCS read access via Application Default Credentials. No OAuth client needed for server-side imports.
📊 Scaling
Min 0 / Max 5 instances · 512 MB RAM · 1 vCPU per instance.
WebSocket sessions are stateful — each revision handles concurrent connections independently.
